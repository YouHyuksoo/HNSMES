# HAENGSUNG HNSMES - 메뉴 트리 구조

**작성일**: 2026-02-06
**참고**: 메뉴는 DB(`PKGSYS_MENU.GET_MENU`)에서 동적 로딩되며, 아래는 소스 코드 기반으로 재구성한 전체 폼 목록입니다.

---

## 메뉴 로딩 아키텍처

```
[Application Start]
    |
    v
[COMLOGIN - 로그인]
    |
    v
[PKGSYS_MENU.GET_MENU / GET_MENUROLE 호출]
    |
    v
[iDATMainMenu.cs - TreeList 메뉴 렌더링]
    |
    +-- NORMAL 모드: NavBarControl (좌측 네비게이션)
    +-- RIBBON 모드: RibbonControl (상단 리본)
    +-- TILE 모드: TileControl (타일 메뉴)
```

**메뉴 데이터 구조**: `MENUSEQ(PK) / UPRSEQ(부모) / MENUNAME(표시명) / FORM(폼클래스) / FORMROLE(권한) / IMGIDX(아이콘)`

## 폼 ID 명명 규칙

```
[MODULE][TYPE][NUMBER]
   │      │      │
   │      │      └── 201~299: 일련번호
   │      │
   │      └── A: 주요 업무 (등록/처리)
   │          B: 조회/이력/현황
   │          C: 고급/특수 기능
   │          H: 도움말/참조
   │
   └── COM: 공통     MAT: 자재     MNT: 보전
       MST: 기준정보  OSC: 외주     PRD: 생산
       RPT: 리포트    SAL: 영업     SYS: 시스템
       TST: 샘플

PopUp 명명: POP_[MODULE][번호] 또는 [FORMID]_POPUP
```

## 메뉴 권한 구조

```
[사용자 로그인]
    │
    v
[USERROLE 확인] ──> PKGSYS_MENU.GET_MENUROLE(역할별 메뉴)
    │
    v
[메뉴 필터링] ──> FORMROLE 기반 접근 제어
    │
    v
[화면 표시] ──> 권한 있는 메뉴만 TreeList에 렌더링
```

- **SYSA206**: 사용자 등급 마스터에서 역할(Role) 정의
- **SYSA207**: 메뉴 권한 마스터에서 역할별 메뉴 접근권한 설정
- **COMFAVORITES**: 사용자별 즐겨찾기 메뉴 (최대 10개)

## 주요 업무 플로우 (메뉴 기반)

```
[생산 프로세스]
PRDC201 일별생산계획 → PRDA201 작업지시생성 → PRDA202 자재요청
    → PRDA203~222 작업실적등록 → PRDA206 불량등록
    → PRDA208~221 통전/육안검사 → PRDA205 포장등록
    → SALA204 OQC등록 → SALA201~203 출하

[자재 프로세스]
MATA203 자재입고 → MATA201 IQC등록 → MATA202 라벨발행
    → MATA206 위치이동 → MATB201 입출고이력 조회
    → MATA208 재고실사

[기준정보 설정]
MSTA201 공장 → MSTA209 라인 → MSTA205 품목
    → MSTA206 BOM → MSTA207 라우팅 → MSTA208 불량코드
    → MSTA217 압착기준

[시스템 설정]
SYSA211 사원등록 → SYSA206 등급설정 → SYSA205 메뉴등록
    → SYSA207 권한설정 → SYSA203 공통코드 → SYSA201 용어등록
```
